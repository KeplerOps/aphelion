# ADR-005: Preserve a Single-Node Local Storage and Transaction Model in V1

- **Status**: Accepted
- **Date**: 2026-04-05
- **Superseded by**: None

## Context

Kernel already provides single-node storage and transactions. Replacing immediately negates adoption value.

## Decision

V1 preserves single-node local persistent storage and transaction model. No distributed transactions, shared storage, disaggregated storage, or cross-node log shipping. Backup, restore, export, import, crash recovery are in scope.

## Consequences

**Positive:**
- Preserves kernel value -- the existing storage and transaction model is proven and functional.
- Aligns with the first-product scope defined in ADR-003.
- Hardens local durability first before introducing distributed complexity.

**Negative:**
- HA and failover are deferred to later work.
- Some cloud-native storage patterns (disaggregated storage, shared-nothing) are deferred.

**Risks:**
- Later replication work may require intrusive changes to the storage layer.
- V1 storage decisions may be mistaken for permanent architectural commitments.
