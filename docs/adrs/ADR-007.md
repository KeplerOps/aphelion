# ADR-007: Expose the Product as a Server, Not as the Raw Embedded Kernel API

- **Status**: Accepted
- **Date**: 2026-04-05
- **Superseded by**: None

## Context

Product target is a database server, not an internal library. Raw kernel API exposure collapses the boundary between engine internals and product surface.

## Decision

Product exposed as server process with product-owned external interface. Raw kernel API is an internal implementation detail. Auth, audit, and observability live at the server layer. Embedded use for internal tooling and tests is acceptable but is not a product contract.

## Consequences

**Positive:**
- Clean boundary between product surface and engine internals.
- First-class security, audit, and ops capabilities at the server layer.
- Product is not bound to the kernel's API shape or evolution.

**Negative:**
- Adds server implementation work early in the project.
- Independent API surface to maintain alongside the kernel.

**Risks:**
- Convenience shortcuts bypass the server layer during development.
- Poor boundary discipline leaks kernel assumptions into the product API.
