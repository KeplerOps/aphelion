# ADR-001: Bootstrap on a Kuzu Fork as the Initial Engine Kernel

- **Status**: Accepted
- **Date**: 2026-04-05
- **Superseded by**: None

## Context

The team needs a starting point for the engine core. Evaluated: Kuzu (MIT, modern single-node kernel, archived Oct 2025), MillenniumDB (GPL v2, not production-ready), Graphflow (research artifact), greenfield (schedule risk).

## Decision

Use a Kuzu fork as the initial execution and storage kernel. Fork and vendor under project control. The maintained fork lives in the `KeplerOps` GitHub organization as the canonical source for kernel development and release control. Do not adopt MillenniumDB (GPL v2), Graphflow (research), or greenfield (risk).

## Consequences

**Positive:**
- Avoids rebuilding kernel from scratch.
- MIT licensed, no copyleft constraints.
- Strong design work in the existing codebase.
- Preserves replacement option -- the fork is an implementation detail, not the product boundary.

**Negative:**
- Upstream archived (Oct 2025) -- no further upstream contributions.
- Team owns full stewardship of the fork.
- Single-node design does not solve HA requirements.

**Risks:**
- Deep kernel assumptions may conflict with later product needs.
- Fork maintenance burden grows over time.
